## 상세 시나리오

### Q-01. 첫 진입

**준비**: localStorage 전체 초기화 후 진입  
**단계**:
1. `https://cafemapapp.vercel.app/` 접속
2. 홈 화면 로드 확인

**기대 결과**:
- 앱명 표시
- 반경/인원/분위기/조건 선택 UI 표시
- 추천받기 버튼 비활성(또는 활성) 확인
- 즐겨찾기 0개, 최근 본 카페 0개 표시

---

### Q-02. 위치 허용

**단계**:
1. 홈 진입
2. 위치 허용 버튼 클릭
3. 브라우저 위치 권한 팝업에서 "허용"

**기대 결과**:
- 위치 취득 성공 메시지 또는 상태 변경 확인
- 반경 선택 후 추천 가능

---

### Q-03. 위치 거부

**단계**:
1. 홈 진입
2. 위치 권한 팝업에서 "거부"

**기대 결과**:
- fallback 메시지 표시 (거부 안내)
- 앱이 크래시되지 않음
- 다른 기능(인천 BEST, 즐겨찾기 등)은 정상 작동

---

### Q-04. 조건 선택

**단계**:
1. 반경 선택 (1km / 3km / 5km)
2. 인원 선택 (혼자 / 2~4명 / 5명 이상)
3. 분위기 선택 (조용히 / 대화 가능)
4. 태그 다중 선택 (콘센트, 와이파이, 늦게까지 등)

**기대 결과**:
- 선택 상태가 시각적으로 구분됨 (활성/비활성)
- 복수 선택 가능 (태그)

---

### Q-05. 추천 결과 있음

**단계**:
1. 위치 허용 후 반경 5km 선택
2. 조건 없이 추천 받기 클릭

**기대 결과**:
- 추천 카페 카드 1~5개 표시
- 거리, 태그, 추천 이유 표시
- 카드 클릭 시 상세로 이동

---

### Q-06. 추천 결과 없음

**단계**:
1. 위치 거부 상태
2. 반경 1km, 가장 엄격한 조건 선택
3. 추천 받기 클릭

**기대 결과**:
- 빈 상태 메시지 표시 ("조건에 맞는 카페를 찾지 못했어요" 또는 유사)
- 크래시 없음

---

### Q-07. 상세 진입

**단계**:
1. 추천 결과 또는 인천 BEST에서 카페 카드 클릭

**기대 결과**:
- 카페명, 주소, 운영시간, 태그, 카공 적합도 표시
- 즐겨찾기 버튼 표시
- 네이버 지도 버튼 표시
- 카페 제안 링크 표시

---

### Q-08. 네이버 지도 링크 클릭 🔴

**단계**:
1. 카페 상세에서 "네이버 지도로 보기" 버튼 클릭

**기대 결과 (브라우저)**:
- 새 탭에서 네이버 지도 URL 열림
- `target="_blank" rel="noopener noreferrer"` 확인

**기대 결과 (WebView)**:
- 외부 브라우저 또는 앱으로 이동 (WebView 내 열리지 않음)
- 뒤로가기 후 앱 상태 유지

---

### Q-09. 즐겨찾기 추가/삭제

**단계**:
1. 카페 상세에서 즐겨찾기(♡) 버튼 클릭 → 추가
2. 홈에서 즐겨찾기 탭 진입 → 카페 확인
3. 다시 즐겨찾기 버튼 클릭 → 삭제
4. 새로고침 후 즐겨찾기 탭 재확인

**기대 결과**:
- 즐겨찾기 추가/삭제 즉시 반영
- 새로고침 후에도 유지 (localStorage 저장)
- 즐겨찾기 탭 카운트 변경 반영

---

### Q-10. 최근 본 카페 기록

**단계**:
1. 카페 상세 여러 개 방문
2. 홈에서 "최근 본 카페" 탭 진입

**기대 결과**:
- 방문한 카페가 최신 순으로 표시
- 최대 20개
- 목록 초기화 버튼 동작 확인

---

### Q-11. 인천 BEST 구/동 선택

**단계**:
1. 홈에서 "인천 BEST" 진입
2. 구 선택 → 동 선택

**기대 결과**:
- 선택한 구의 동 목록 표시
- 선택한 동의 BEST 카페 1~5개 표시
- 점수 기반 정렬 확인

---

### Q-12. 인천 BEST 검색

**단계**:
1. 인천 BEST에서 검색창에 카페명 입력

**기대 결과**:
- 실시간 필터링 (입력 중)
- 매칭 없을 때 빈 상태 표시

---

### Q-13. 카페 제안 제출

**단계**:
1. 홈 또는 카페 상세에서 "카페 제안하기" 진입
2. 카페명, 주소 입력
3. 추천 이유, 태그 선택 (선택)
4. 제출 버튼 클릭

**기대 결과**:
- 필수 항목 미입력 시 에러 메시지
- 제출 완료 화면 표시
- localStorage에 제안 저장 확인

---

### Q-14. 관리자 검수

**접근**: `/?mode=admin`  
**단계**:
1. Q-13에서 제안 제출 후 관리자 페이지 접근
2. 목록에서 방금 제안한 항목 확인
3. 카드 클릭 → 상세 펼치기
4. 승인 또는 반려 클릭

**기대 결과**:
- 제안 항목이 목록에 표시됨 (상태: 대기)
- 승인/반려 후 상태 변경 확인
- 상태 필터 동작 확인
- 일반 사용자 navStack에서 이 화면에 접근할 수 없음

---

### Q-15. 서비스 안내 화면

**단계**:
1. 홈 하단 "서비스 안내" 링크 클릭

**기대 결과**:
- 위치 정보, 저장 데이터, 외부 링크, 카공 기준 섹션 표시
- 뒤로가기로 홈 복귀

---

### Q-16. Android 뒤로가기 🔴

**단계**:
1. 홈 → 인천 BEST → 카페 상세 순으로 진입
2. Android 하드웨어 뒤로가기 버튼 3회 누르기

**기대 결과**:
- 상세 → 인천 BEST → 홈 순으로 복귀
- 홈에서 한 번 더 누르면 앱 종료(또는 WebView 닫힘)
- 이중 pop 발생하지 않음

---

### Q-17. iOS Safe Area 🔴

**단계**:
1. iPhone에서 앱 진입
2. 노치/다이나믹 아일랜드 영역 확인

**기대 결과**:
- 상단 Safe Area 내에 버튼이 가려지지 않음
- 하단 홈 인디케이터와 겹치지 않음

---

### Q-18. 외부 링크 WebView 동작 🔴

**단계**:
1. 앱인토스 WebView에서 네이버 지도 링크 클릭

**기대 결과**:
- 외부 브라우저로 이동 또는 네이버 지도 앱 실행
- WebView 내에서 네이버 지도가 열리지 않음
- 뒤로가기 후 앱 정상 복귀

---

### Q-19. 오류 fallback

**단계**:
1. DevTools에서 localStorage 비활성화 또는 시크릿 모드 접근

**기대 결과**:
- 앱이 크래시되지 않음
- 즐겨찾기/최근 본 카페가 빈 상태로 표시
- safeStorage가 예외를 흡수

---

### Q-20. localStorage 비정상 데이터

**단계**:
1. DevTools > Application > localStorage에서 `kagong_favorites_...` 키에 비정상 값 직접 입력 (예: `"invalid"`, `123`, `null`)
2. 앱 새로고침

**기대 결과**:
- 앱이 크래시되지 않음
- 비정상 데이터는 fallback(빈 배열)으로 처리됨
- 정상 기능 유지
