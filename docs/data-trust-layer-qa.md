## 3. 추천 경험 개선 현황

### 개선된 것

- **운영자 추천 / 조건 기반 추천 섹션 분리** 구현 완료
- **curatorNote placeholder 제거** — 모든 카페에 "자동 수집..." 노출 문제 해결
- **추천 없음 fallback** — 반경 확장, 조건 완화 인라인 제안 추가
- **override 구조** — 운영자 확인 후 즉시 적용 가능한 안전한 패치 구조

### 아직 미완성 (운영자 작업 필요)

- **curated 카페 0개**: 운영자가 docs/curated-candidate-list.md의 20개 후보를 직접 확인 후 curatedOverrides.ts에 등록해야 함
- **점수 모두 3/5 문제**: 631개 카페 속성 점수 미분화 → 추천이 여전히 거리순 편향
- **openHoursSummary 미입력**: 681개 카페가 "방문 전 직접 확인" placeholder 상태

## 6. Phase 18 (앱인토스 제출) 반영 문구

> "카공 어디가? 인천편은 690여 개의 인천 카페를 직접 수집하고, 운영자가 카공 적합성을 기준으로 검수한 카페를 우선 추천합니다. 위치, 분위기, 콘센트/와이파이 조건에 맞는 카페를 TOP 3~5로 좁혀 선택 피로를 줄여드려요."
