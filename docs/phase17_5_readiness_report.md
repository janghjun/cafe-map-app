## 2. 레퍼런스 인사이트 반영 점검

### 2-1. 추천 결과에 위치 감각이 충분한가?

**현재**: 카드에 `{district} {dong}` + 거리 레이블 표시.  
**부족한 점**: 지도 미도입이지만 동 이름만으로 사용자가 방향을 인지하기 어렵다.  
**평가**: △ 기능적으로는 작동하나 "송도" "청라" 같은 랜드마크 지명이 거리보다 직관적으로 느껴질 수 있음. 방향 힌트 없음.

### 2-2. 카페가 실제 존재하는지/검증되었는지 알 수 있는가?

**현재**: 데이터는 `status: "active"` 필터만 적용. 상세 화면에서 검증 여부 노출 없음.  
**부족한 점**: 사용자가 데이터가 언제 기준인지 알 방법이 없음. "거지맵·두쫀쿠맵"처럼 "운영팀이 확인한 카페예요" 같은 신뢰 신호 없음.  
**평가**: ✗ 신뢰 레이어 완전 부재.

### 2-3. 마지막 확인일/최신성 표시가 있는가?

**현재**: `updatedAt` 필드가 타입과 mock 데이터에 존재하지만, UI에 표시되지 않음. 모든 mock 카페가 `2025-01-01` 고정값.  
**부족한 점**: 사용자는 데이터가 언제 수집되었는지 전혀 알 수 없음.  
**평가**: ✗ 존재하는 필드를 UI로 노출하지 않음.

### 2-4. 카페 제안이 신뢰 가능한 검수 흐름으로 보이는가?

**현재**: 제출 완료 화면은 있음. "운영팀이 확인하고 검수를 통과하면 반영"이라는 서비스 안내 텍스트 있음.  
**부족한 점**: 
- 제안 완료 후 사용자에게 "검수 이후 반영됩니다"라는 명시적 메시지는 제출 완료 화면에 포함되어 있는지 확인 필요
- 제안 후 어떤 과정을 거치는지 UX 흐름이 모호함.  
**평가**: △ 내부 구조는 있으나 사용자 신뢰 표현 부족.

### 2-5. FAQ/서비스 안내가 충분한가?

**현재**: `ServiceInfoPage` 5섹션 (위치/저장/제안/외부링크/카공기준).  
**부족한 점**:
- "이 카페 정보는 언제 기준인가요?" 같은 FAQ가 없음
- "폐업/이전 시 어떻게 하나요?" 안내 없음
- "카공 적합도는 어떻게 측정하나요?" 상세 설명 없음  
**평가**: △ 기본 내용은 있으나 사용자 핵심 질문에 답하지 못함.

### 2-6. 지도 앱처럼 보이지 않으면서 지도형 레퍼런스의 장점을 흡수하고 있는가?

**현재**: "지도 앱" 아님 명확. 거리 레이블로 근접성 표현.  
**부족한 점**: 지도형 레퍼런스의 핵심인 "큐레이션 + 검증 + 최신성"을 홈/결과/상세에서 적극 표현하지 않음.  
**평가**: △ 방향은 맞으나 "직접 검증된 카공 카페"라는 브랜드 메시지 약함.

### 2-7. 외부 리뷰/평점/사진 복제 리스크가 없는가?

**현재**: 외부 리뷰 원문 없음. 모든 적합도 점수는 자체 기준.  
**평가**: ✅ 리스크 없음. 서비스 안내에도 명시되어 있음.

## 3. Must 개선 항목

> 앱인토스 심사 전 반드시 보강해야 함. 기능 추가가 아닌 기존 데이터·텍스트 개선.

### M-1. 카페 상세에 "마지막 확인일" 표시

- `updatedAt` 필드가 이미 있음 — 상세 하단에 "정보 기준: YYYY.MM" 형식으로 표시
- 사용자가 데이터 최신성을 인지할 수 있는 최소한의 신뢰 신호
- mock 데이터의 `updatedAt` 을 실제적인 날짜로 업데이트 필요

### M-2. 카페 상세에 "검증 상태" 뱃지 추가

- `status: "active"` 카페에 "운영 확인됨" 텍스트 또는 소형 뱃지 표시
- "운영팀이 직접 확인한 카페예요"라는 신뢰 레이어
- 코드 변경 최소 — 상세 화면 기본 정보 섹션에 1줄 추가

### M-3. 서비스 안내에 핵심 FAQ 추가

현재 5섹션에 2개 FAQ 섹션 추가:
- "카페 정보는 언제 기준인가요?" — 수동 수집 + 주기적 검증 + 방문 전 확인 권고
- "폐업/이전한 카페가 있다면?" — 수정 제안 기능 안내 + 검수 반영 프로세스

## 4. Should 개선 항목

> 심사 완성도를 높이지만 없어도 제출 가능. 1~2일 내 구현 가능한 범위.

### S-1. 추천 결과 요약 문구에 지역 감각 추가

현재: "내 주변 5km 안에서 조건에 맞는 카페 4곳을 찾았어요"  
개선: "연수구·남동구 주변 5km에서 카공 카페 4곳을 찾았어요" (결과 카페의 district 기반으로 동적 생성)

### S-2. 카페 제안 완료 화면에 검수 프로세스 안내 강화

현재: 완료 메시지만 있는 것으로 추정  
개선: "제안해 주셔서 감사해요. 운영팀이 카페를 직접 확인한 뒤 앱에 반영할게요." + 예상 기간 안내

### S-3. 인천 BEST 화면에 "큐레이션 기준" 안내 추가

현재: 점수 기준으로만 정렬  
개선: "이 목록은 카공 적합도 기준으로 선정된 카페예요" 한 줄 안내 — 네이버 별점 기반이 아님을 명확히

### S-4. 홈 화면 한 줄 설명 개선

현재: 없거나 일반적인 설명  
개선: "인천 카공 카페를 직접 검증했어요 — 조용함, 콘센트, 와이파이 기준" 같은 차별화 포인트 표현
