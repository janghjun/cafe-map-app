## 4. 핵심 설계 결정

### 4.1 navStack 내비게이션

React Router를 쓰지 않고 `NavState` 타입의 배열을 직접 관리한다.

```ts
type NavState =
  | { page: "home" }
  | { page: "recommendations"; preference: UserPreference; userLocation: Coords }
  | { page: "cafeDetail"; cafe: Cafe; distanceLabel?: string }
  | { page: "districtBest" }
  | { page: "favorites" }
  | { page: "recentViews" }
  | { page: "suggestCafe" };
```

- `push(state)` → 화면 이동
- `pop()` → 뒤로가기
- discriminated union으로 각 페이지 props를 타입 안전하게 전달

### 4.2 추천 점수 (100점 기준)

`src/utils/recommendation.ts`의 `scoreCafeByPreference` 함수:

| 항목 | 배점 |
|---|---|
| 거리 적합도 | 30점 |
| 인원 적합도 (solo / group) | 20점 |
| 분위기 적합도 (quiet / talkable) | 15점 |
| 콘센트 | 10점 |
| 체류 적합성 | 10점 |
| 와이파이 | 5점 |
| 영업시간/24시간 | 5점 |
| 커피/디저트 | 5점 |

- 조건을 선택하지 않은 항목은 만점 부여 (페널티 없음)
- 기본 TOP3 노출, 더보기 시 TOP5

### 4.3 localStorage 안전 접근 (`safeStorage.ts`)

모든 서비스가 직접 `localStorage`를 쓰지 않고 `safeGet / safeSet / safeRemove`를 경유한다.

- `safeGet<T>(key, fallback, validate?)`: JSON 파싱 실패, 타입 불일치, 저장소 비가용 시 fallback 반환
- `safeSet`: `QuotaExceededError` / `SecurityError` 조용히 흡수
- `isStringArray`: string[] 런타임 타입 검증 (즐겨찾기, 최근본 카페에 사용)

### 4.4 외부 지도 링크

`src/utils/naverMap.ts`의 `getCafeMapUrl`:

1. `cafe.naverMapUrl`이 있으면 그대로 사용
2. 없으면 `이름+주소`를 `encodeURIComponent`로 인코딩한 네이버 지도 검색 URL 생성

nmap:// 앱 스킴 및 lat/lng 기반 길찾기 URL은 WebView 정책 불확실로 제거됨.
